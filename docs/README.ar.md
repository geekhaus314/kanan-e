# KananOS — متجرك الإلكتروني للبيع بالجملة

> **شركتك:** United Distribution (فلوريسانت، ميسوري)  
> **موقعك:** united.kananos.com  
> **بُني بواسطة:** KananOS (منصتك الخاصة)

---

## ما هذا المشروع؟

KananOS هو نظام متجر إلكتروني خاص بك. تخيل أن لديك أمازون أو علي بابا خاص بك — ولكن لمتجرك فقط في البداية. United Distribution هي أول متجر على هذه المنصة.

**طريقتان للشراء منك:**
1. **التجزئة (Retail)** — أي شخص عمره فوق ٢١ سنة يستطيع الشراء (بعد التحقق من العمر)
2. **الجملة (Wholesale)** — الشركات التي لديها رخصة تبغ تحصل على أسعار خاصة

## كيف المنظمة؟

```
kananos/
├── apps/
│   └── web/              ← موقعك الفعلي (united.kananos.com)
├── packages/
│   ├── database/         ← مخزن المنتجات والعملاء والطلبات
│   ├── validations/      ← قواعد فحص البيانات (مثال: "البريد الإلكتروني صحيح")
│   └── config/           ← إعدادات عامة
├── docker-compose.yml    ← ملف لتشغيل قاعدة البيانات على جهازك
└── docs/
    ├── README.en.md      ← الوثائق بالإنجليزية
    └── README.ar.md      ← هذا الملف (العربية)
```

### كيف يصل العملاء إلى متجرك؟

عندما يكتب أحدهم `united.kananos.com` في المتصفح، يظهر له متجر **أنت** تلقائياً. لو انضمت متاجر أخرى للمنصة لاحقاً، كل واحد يكون له عنوان مثل `otherstore.kananos.com`.

| الرابط | ماذا يرون؟ |
|--------|-----------|
| `united.kananos.com` | الصفحة الرئيسية لمتجرك |
| `united.kananos.com/products` | قائمة منتجاتك |
| `united.kananos.com/cart` | سلة التسوق الخاصة بهم |
| `united.kananos.com/account` | طلباتهم السابقة |
| `united.kananos.com/admin` | لوحة تحكمك أنت (المشرف) |

## تشغيل الموقع

### الخطوة ١: ماذا تحتاج؟

- أي كمبيوتر (لابتوب أو مكتبي)
- اتصال بالإنترنت
- حساب مجاني على **Vercel** (مكان استضافة المواقع)
- حساب مجاني على **Neon** (مكان تخزين البيانات)

### الخطوة ٢: المفاتيح السرية

ملف `.env.local` يحتوي على جميع مفاتيحك السرية — مثل كلمات المرور التي تسمح للخدمات المختلفة بالتحدث مع بعضها. هذه كلها جاهزة. **لا تشاركها مع أي أحد.**

### الخطوة ٣: التشغيل على جهازك (للتجربة)

```bash
docker compose up -d     # شغّل قاعدة البيانات (مثل تشغيل السيارة)
pnpm install             # حمّل كل القطع التي يحتاجها الموقع
pnpm db:push             # أنشئ جداول قاعدة البيانات (رفوف لبياناتك)
pnpm db:seed             # ضع منتجات تجريبية لترى كيف يبدو الموقع
pnpm dev                 # شغّل الموقع على جهازك
```

ثم افتح `http://localhost:3000/united` في متصفحك.

### الخطوة ٤: النشر المباشر (على الإنترنت)

مبرمجك سيقوم بـ:
1. ربط Vercel بهذا الكود
2. إعداد قاعدة البيانات على Neon
3. ربط `united.kananos.com` بـ Vercel
4. تشغيل أوامر إعداد البيانات
5. موقعك يعمل على الإنترنت!

## اللغات في موقعك

### من يرى أي لغة؟

| الشخص | اللغة |
|-------|-------|
| **أنت** (عند دخولك لوحة الإدارة) | العربية |
| **العميل** يتصفح متجرك | لغة متصفحه (تلقائياً) |
| **مدير المنصة** (أنت، للصيانة) | الإنجليزية |

الزبائن يستطيعون تغيير اللغة بأي وقت عبر زر اللغة في أعلى الصفحة.

### كيف يتم اختيار اللغة؟

1. لو ضغط العميل على زر لغة معينة، يحفظ اختياره تلقائياً
2. لو لم يختر، يفحص الموقع إعدادات لغة المتصفح
3. لو لم تتطابق مع العربية أو الإنجليزية، يفحص موقعه (عنوان IP)
4. لو مازال لا شيء، الإنجليزية هي الاختيار الافتراضي

## لوحة التحكم الخاصة بك

**الرابط:** `united.kananos.com/admin`  
**اللغة:** العربية (افتراضياً لك)

### ماذا يمكنك أن تفعل؟

| القسم | ماذا يحتوي؟ |
|-------|------------|
| **لوحة التحكم** | نظرة سريعة: عدد المنتجات، طلبات التحقق المعلقة، أحدث الطلبات |
| **التحقق من العمر** | العملاء يرفعون هوياتهم لإثبات أن عمرهم فوق ٢١. أنت توافق أو ترفض. |
| **الطلبات** *(قريباً)* | عرض وإدارة طلبات العملاء |
| **المنتجات** *(قريباً)* | إضافة أو تعديل أو حذف المنتجات |
| **موافقات الجملة** *(قريباً)* | مراجعة رخص التبغ لعملاء الجملة |

### كيف تسجل الدخول؟

مبرمجك أنشأ لك حساب مشرف:
- **البريد الإلكتروني:** `admin@uniteddistribution.com`
- **كلمة المرور:** (اسأل مبرمجك)

بعد تسجيل الدخول، سترى لوحة التحكم بالأرقام والأزرار.

## كيف يشتري العملاء منك؟

### عميل عادي (تجزئة)

1. يذهب إلى `united.kananos.com`
2. يتصفح المنتجات، يضغط على منتج ليرى التفاصيل
3. لو كان تبغ أو فيب، يجب إدخال تاريخ الميلاد ورفع الهوية
4. أنت (أو فريقك) تراجع الهوية في لوحة التحكم
5. بعد الموافقة، يقدر يضيف إلى السلة
6. يكتب عنوان الشحن
7. يقدم الطلب
8. تتصل به أو ترسل له إيميل لأخذ الدفع (نظام الدفع قيد التطوير)
9. تشحن المنتجات

### عميل شركة (جملة)

1. نفس الخطوات أعلاه، ولكن أيضاً:
2. يذهب إلى صفحة التسجيل للبيع بالجملة
3. يكتب اسم الشركة، الرقم الضريبي، ورخصة التبغ
4. أنت تراجع الرخصة وتوافق عليها في لوحة التحكم
5. يشوف أسعار الجملة تلقائياً (أقل من التجزئة)
6. يقدر يطلب عروض أسعار للطلبات الكبيرة

## قاعدة البيانات (أين يتم تخزين كل شيء)

تخيل قاعدة البيانات مثل خزانة ملفات مع أدراج مصنفة:

| الدرج (الجدول) | ماذا بداخله؟ |
|----------------|-------------|
| `tenants` | المتاجر التي تستخدم المنصة (حالياً United فقط) |
| `users` | كل من لديه حساب (عملاء + أنت) |
| `tenant_users` | أي العملاء ينتمون لمتجرك، ودورهم |
| `products` | كل منتج تبيعه (كود المنتج، الاسم، السعر، المخزون) |
| `categories` | مجموعات المنتجات (فيب، تبغ، اكسسوارات...) |
| `bulk_pricing_tiers` | أسعار مخفضة عند الشراء بالجملة |
| `cart_items` | المنتجات التي وضعها العملاء في السلة قبل الشراء |
| `orders` | الطلبات المكتملة من العملاء |
| `order_items` | المنتجات المحددة داخل كل طلب |
| `age_verifications` | هويات العملاء المرفوعة وحالتها |
| `quote_requests` | طلبات عروض أسعار من عملاء الجملة |

## التقنيات المستخدمة (شرح بسيط)

| الجزء | ما هو؟ |
|-------|--------|
| **محرك الموقع** | Next.js — العقل الذي يظهر الصفحات |
| **لغة البرمجة** | TypeScript — اللغة التي كُتب بها الكود |
| **قاعدة البيانات** | PostgreSQL — مخزن جميع بياناتك |
| **نظام الدخول** | Auth.js — يحمي حسابات العملاء |
| **التنسيق** | Tailwind — يجعل الموقع يبدو احترافياً |
| **البحث** | Meilisearch — يشتغل على شريط البحث |
| **تخزين الملفات** | Cloudflare R2 — يحفظ الصور والهويات المرفوعة |
| **البريد الإلكتروني** | Resend — يرسل تأكيدات الطلبات والإشعارات |
| **الدفع** | NMI — يعالج دفعات البطاقات الائتمانية (قيد التجهيز) |
| **صور المنتجات** | Cloudflare R2 — مكان حفظ صور المنتجات |
